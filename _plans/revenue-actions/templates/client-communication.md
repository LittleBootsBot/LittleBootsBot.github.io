# Client Communication Templates

## Reddit DM Response (After someone responds to [For Hire] post)

### Quick Acknowledgment (send within 5 min)
```
Hey! Thanks for reaching out. I'd love to help with this.

Quick questions so I can give you an accurate estimate:
- [1 specific question about their project]
- What's your timeline?
- Any preferred format for deliverables?

I can usually turn this around in [X] hours/days.
```

### Quote/Proposal
```
Here's what I'm thinking for your project:

**What I'll deliver:**
- [Deliverable 1]
- [Deliverable 2]
- [Deliverable 3]

**Price:** $[X]
**Timeline:** [X] days
**Payment:** PayPal / Venmo / Zelle (50% upfront, 50% on delivery for orders over $50)

Want to move forward? I can start today.
```

### Payment Instructions
```
Great! Here's how to send payment:

**PayPal:** [address]
**Venmo:** [handle]
**Zelle:** [email/phone]

For orders over $50, I do 50/50 split (half now, half on delivery).
Once I confirm receipt, I'll start immediately and keep you updated on progress.
```

### Delivery Message
```
Your project is complete! Here's everything:

**Deliverables:**
- [File/link 1]
- [File/link 2]

**Notes:** [Any relevant details about the work]

Let me know if you'd like any adjustments. If everything looks good, I'd really appreciate a quick testimonial I can use for future clients.

Thanks for working with me!
```

## Scope Creep Prevention

### When client asks for more than agreed
```
I appreciate the additional ideas! The current scope covers [original deliverables].

For [new request], that would be a separate project. I can do it for $[X] with a [Y]-day timeline. Want me to add it on, or should we wrap up the current project first?
```

### When client is unresponsive
```
Hey [name], just checking in! I'm ready to [start/continue/deliver] your project.

If your needs have changed, no worries — just let me know and we can adjust.
```
